My Software Design Process

The design process is dynamic and iterative with the end goal of ensuring that a product is accessible, functional, durable and simple for uptake. By performing primary and secondary research, we may design prototypes that fit the current needs of users and incorporate elements of anticipated future trends.

Understand User Needs

Research is conducted by discussing expectations/requirements with end users and stakeholders and analyzing current market trajectories. Then, we can create a UI design based on direct and implicated preferences. The wireframing process incorporates suggestions as well as follows intuitive workflows present in other applications, such as swiping right/up or hitting cancel at the top right.

Create Prototype

The implementation of the design by the software or media team is performed. I like to have an active (if possible, hands-on) role in the developmental life cycle, as I can also speak the language of computer programmers. Most design elements can be customized as widgets, but can be modified if software tools do not support the design.

Test & Revise

User studies and beta testing are essential steps of UX and UI research as user satisfaction generally ensures the product launch will be successful. Observational or qualitative trends may indicate that aspects of the design need to be modified, and therefore executed.

Project 1: Tangible Chromatin

TC is a visualization platform for biologists to store and display large amounts of experimental data, mainly from chromatin fibers. It was a collaboration between the Sabatinos Lab and the synlab.

Challenge: How do we display process and store from large and diverse chromatin fiber datasets?

Solution: Create an interactive system in which users may visualize and compare their data tangibly using their mobile phones.

My roles:

  • + Research the needs of biologists, such as the quantity of their data and analysis expectations, and assess design strategy with regards to our goal.
  • + Liasion between biologists (end users) and engineering/software team to inform choice architecture, such as selection parameters (e.g. maximum 2 proteins) and exclusion/inclusion criteria.
  • + Create the concept to process to threshold the data, implemented as an R back-end.
  • + Conduct user studies (interviews and questionnaires) to observe user experience navigating new technology and address concerns/feedback.

Project 2: Hourglass

Hourglass is a user-friendly tool created for pathologists with multiparameteric bioimaging datasets.

Challenge: How can information can be mined from these datasets in a streamlined approach?

Solution: Design and implement a downloadable desktop application, in which users may easily upload their data without privacy concerns and attain results the same day.

My roles:

  • + Conduct user research to understand how a user interface may support exploration. Key finding: default parameters needed for newer data analysts, while heavily customizable advanced options for more experienced users.
  • + Create blueprint of sequential tab layout corresponding to simple workflow steps based on user preferences, updated with subsequent user studies in iterative research methodology.
  • + Consistently anticipated potential issues and tried to address them. However executation was often limited by software limitations, thus workarounds were incorporated into the original design.

Project 3: MEC Explorer

Mammary epithelial cell (MEC) explorer is an interactive bioinformatics web-application created to mine single-cell and proteome datasets.

Challenge: Which is the most direct manner for users to explore genes of interest?

Solution: Design a web tool that allows query of any gene and upload data to web server.

My roles:

  • + Empathize with users and understand the expectations of a tool to facilitate easy and customizable data analysis.
  • + Create a simple, quick to use UI - so I decided on 1 page per dataset (total: 2), with consistent layout (left - user input, right - output).
  • + Original design offered a visualization tool and download button for plot only. However during testing, users indicated that accompany transformed data would be beneficial. I modified the design and added functionality to download data as well.
